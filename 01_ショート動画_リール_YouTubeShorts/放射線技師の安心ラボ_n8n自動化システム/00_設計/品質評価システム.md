# n8n自動化 品質評価システム

## 目的

自動化システムが実運用に耐えられるかを自己採点し、120点に近づくまで改善する。

## 採点項目

各10点満点で採点する。

| 項目 | 点数 | 評価ポイント |
|---|---:|---|
| 自動化率 | /10 | 手作業をどれだけ減らせるか |
| 量産性 | /10 | 週3〜5本を回せるか |
| 再利用性 | /10 | 成功パターンを使い回せるか |
| 初心者理解度 | /10 | 初めてでも組めるか |
| 保守性 | /10 | 修正しやすいか |
| エラー耐性 | /10 | 失敗時に止まらないか |
| 拡張性 | /10 | ブログ・LINEにも広げられるか |
| コスト効率 | /10 | 無駄なAPI実行が少ないか |
| 作業削減 | /10 | 実作業時間を減らせるか |
| 実運用性 | /10 | 現実に運用できるか |
| 合計 | /100 |  |

## 判定基準

| 点数 | 判定 | 対応 |
|---:|---|---|
| 80点未満 | 改善 | ワークフローを単純化 |
| 90点以上 | 実用可能 | 運用開始可能 |
| 100点 | 強い | 安定運用候補 |
| 120点 | 半自動工場レベル | ボーナス込みの理想値 |

## 120点評価について

基本採点は100点満点。

以下のボーナスが強い場合、最大20点を加点する。

| ボーナス | 最大点 |
|---|---:|
| 成功パターンを自動で次回へ反映できる | +5 |
| エラー管理が自動化されている | +5 |
| コスト削減設計がある | +5 |
| リール・ブログ・LINEを横断できる | +5 |
