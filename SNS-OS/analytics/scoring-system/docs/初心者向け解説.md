# 初心者向け解説

## このシステムで分かること

投稿前に「このリールは伸びそうか」を100点満点で確認できます。

点数が低い場合は、投稿前にフック、サムネ、保存誘導、コメント誘導を直します。

## まず見るべき点

### 1. フック力

最初の1〜3秒で見たいと思えるか。

弱い例:

```text
今日はCTについて説明します
```

強い例:

```text
CTが怖い人、これだけ知ってください
```

### 2. 保存率期待

後で見返す理由があるか。

強い例:

```text
検査前に見返せるよう保存してね
```

### 3. コメント誘導力

答えやすい質問になっているか。

強い例:

```text
CTとMRI、どっちが怖いですか？
```

### 4. 危険ポイント

医療系では、強すぎる断定に注意します。

避けたい例:

```text
絶対安全です
絶対危険です
これで診断できます
```

おすすめ:

```text
必要な検査なら過度に怖がりすぎなくて大丈夫です
不安な時は医師やスタッフに相談してください
```

## 点数の見方

| 点数 | 判断 |
|---:|---|
| 90〜100 | 投稿候補。かなり強い |
| 75〜89 | 投稿可能。少し改善するとよい |
| 60〜74 | 改善推奨 |
| 59以下 | 作り直し |

## 運用の流れ

1. 台本を作る
2. 評価テンプレで手動採点する
3. 可能なら `score_engine.py` で自動採点する
4. 改善提案を反映する
5. A/Bテスト案からサムネやフックを2案作る
6. 投稿後に実際の保存率と比べる
